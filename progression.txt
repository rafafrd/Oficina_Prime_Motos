PROGRESSAO — Sistema Oficina Prime Motos
========================================

ETAPA 1 — ANALISE DA BASE
- Base original identificada como projeto estático sem back-end real.
- Arquivos iniciais: `.gitignore`, `index.html`, `style.css`, `script.js`, `progression.txt`.
- Decisão: preservar apenas a referência visual de tema escuro e reconstruir a aplicação em MVC.

ETAPA 2 — ARQUITETURA FINAL
- Criação de `server.js` e `src/app.js`.
- Definição clara de MVC com `controllers`, `models`, `routes`, `middlewares`, `services`, `utils`.
- Front servido pelo Express com páginas protegidas por sessão.
- HTMLs movidos para `public/` e assets separados por responsabilidade.

ETAPA 3 — BANCO SQLITE
- Implementação de `src/schema.sql` com 12 tabelas.
- Implementação de `src/seed.sql` com usuários seed para cliente, funcionário, admin e fornecedor.
- Criação de inicialização automática do banco em `src/database/init.js`.
- Inclusão de script `db:reset` e bancos isolados para testes.

ETAPA 4 — AUTENTICACAO E AUTORIZACAO
- Login com sessão usando `express-session`.
- Hash de senha com `bcryptjs`.
- Middleware de autenticação e autorização por papel.
- Rotas protegidas para cliente, funcionário e fornecedor.

ETAPA 5 — MODULOS DE NEGOCIO
- Ordens de serviço com fluxo obrigatório de status.
- Histórico de atualizações.
- Agenda integrada com reagendamento.
- Gestão de peças, mão de obra, orçamento e total calculado.
- Portal específico do fornecedor com recursos permitidos.

ETAPA 6 — FRONT-END FUNCIONAL
- Tela de login real.
- Dashboard com métricas.
- Página de ordens com timeline, histórico e formulários por papel.
- Página de agenda.
- Página do fornecedor.
- Página de relatório com exportação PDF client-side.
- UI refeita em modo escuro técnico com destaque laranja/coral.

ETAPA 7 — TESTES AUTOMATIZADOS
- API/login com Jest + Supertest.
- Interface com Playwright.
- Cobertura mínima exigida implementada para autenticação, ordens, histórico, cálculo, navegação e PDF.

ETAPA 8 — DOCUMENTACAO E ENTREGA
- README completo.
- TESTES.md com roteiro prático.
- `.gitignore` atualizado.
- Projeto preparado para execução local com Node.js, Express e SQLite.
